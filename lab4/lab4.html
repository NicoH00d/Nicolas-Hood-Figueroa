¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Diferencias:

Origen y Propósito: Java es para aplicaciones empresariales, mientras que JavaScript es para agregar interactividad a páginas web.

Ejecución: Java es compilado y ejecutado en una máquina virtual, mientras que JavaScript es interpretado en el navegador.

Tipado: Java es fuertemente tipado y estático, mientras que JavaScript es débilmente tipado y dinámico.

Sintaxis y Semántica: Java es más rígido y orientado a objetos, mientras que JavaScript es más flexible y combina orientación a objetos con programación funcional y prototípica.

Aplicaciones: Java es ampliamente utilizado en aplicaciones de escritorio, móviles, empresariales y sistemas integrados, mientras que JavaScript es para desarrollo web del lado del cliente y del servidor, aplicaciones móviles híbridas y juegos.

Similitudes:

Sintaxis Básica: Comparten algunas similitudes en la sintaxis básica.

Ecosistema: Ambos tienen un ecosistema amplio y activo de herramientas y bibliotecas.

Programación Orientada a Objetos: Ambos admiten la programación orientada a objetos.

vent-Driven: Ambos son compatibles con la programación orientada a eventos para crear aplicaciones interactivas.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)

El objeto Date en JavaScript proporciona varios métodos para trabajar con fechas y horas. Aquí hay al menos cinco de los métodos más comunes:

getDate(): Devuelve el día del mes (del 1 al 31) para la fecha especificada.

getMonth(): Devuelve el mes (del 0 al 11) para la fecha especificada.

getFullYear(): Devuelve el año (como un número de cuatro dígitos) para la fecha especificada.

getHours(): Devuelve la hora (0-23) para la fecha especificada.

getMinutes(): Devuelve los minutos (0-59) para la fecha especificada.

¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

push(): Este método agrega uno o más elementos al final de un array y devuelve la nueva longitud del array.

pop(): El método pop() elimina el último elemento de un array y devuelve ese elemento.

splice(): Este método cambia el contenido de un array eliminando o reemplazando elementos existentes y/o agregando nuevos elementos en su lugar.

slice(): slice() devuelve una copia superficial de una porción de un array en un nuevo array.

indexOf(): Este método devuelve el primer índice en el que se encuentra un elemento dado en el array, o -1 si no se encuentra.

¿Cómo se declara una variable con alcance local dentro de una función?

Para declarar una variable con alcance local dentro de una función en JavaScript, se utiliza la palabra clave let o const seguida del nombre de la variable dentro del cuerpo de la función. Las variables con alcance local son útiles para limitar la visibilidad y el acceso a las variables dentro de un contexto específico, en este caso, dentro de una función.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Utilizar variables globales dentro de funciones puede tener implicaciones significativas en la legibilidad, mantenibilidad y escalabilidad del código. Las variables globales pueden aumentar el riesgo de colisión de nombres, dificultar el seguimiento de cambios, introducir dependencias ocultas, complicar las pruebas y dificultar la refactorización del código. Para mitigar estos problemas, es recomendable minimizar el uso de variables globales y en su lugar utilizar técnicas que limiten el alcance de las variables, como pasar variables como argumentos de función o utilizar patrones de diseño como el patrón de módulo para encapsular el estado y la funcionalidad relacionados en un ámbito más limitado. Esto puede mejorar la claridad, mantenibilidad y extensibilidad del código a lo largo del tiempo.
