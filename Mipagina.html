Sección de Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la infraestructura subyacente que permite la conexión y la comunicación entre dispositivos, mientras que la WWW es un servicio específico que utiliza Internet para proporcionar acceso a información en línea mediante hipervínculos. Internet es más amplio y abarca una variedad de servicios y tecnologías más allá de la WWW, como el correo electrónico, la transferencia de archivos, las redes privadas virtuales (VPN), las redes sociales y más.

¿Cuáles son las partes de un URL?

Esquema (Scheme): Indica el protocolo de comunicación que se utilizará para acceder al recurso, como "http://" o "https://".

Nombre de dominio (Domain Name): Identifica la ubicación del recurso en Internet e incluye el subdominio (opcional), el nombre de dominio y el dominio de nivel superior (TLD).

Ruta (Path): Especifica la ubicación específica del recurso en el servidor.

Parámetros (Query Parameters): Son pares de clave-valor utilizados para pasar información adicional al servidor. Se separan del resto del URL por un signo de interrogación "?" y los pares de clave-valor están separados por un signo de ampersand "&".

Fragmento (Fragment): Especifica una ubicación específica dentro del recurso, como un identificador o un ancla. Se indica con un signo de numeral "#" seguido de un identificador.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Los métodos HTTP son acciones que indican al servidor qué operación se debe realizar en un recurso determinado.

GET: Se utiliza para solicitar datos de un recurso específico del servidor. Es el método más comúnmente utilizado en navegadores web cuando visitas una página o haces clic en un enlace. Los datos solicitados son devueltos en el cuerpo de la respuesta.

HEAD: Es similar a GET, pero el servidor solo devuelve encabezados de respuesta, sin el cuerpo del recurso. Se utiliza para obtener información sobre el recurso, como el tamaño o el tipo de contenido, sin descargar el recurso completo.

POST: Se utiliza para enviar datos al servidor para que sean procesados, generalmente como parte de un formulario HTML o para realizar una acción que puede tener un efecto secundario, como crear un nuevo recurso en el servidor. Los datos enviados con POST se incluyen en el cuerpo de la solicitud HTTP.

PUT: Se utiliza para enviar datos al servidor para que sean almacenados o actualizados en un recurso específico. PUT reemplaza completamente el contenido del recurso con los datos proporcionados en el cuerpo de la solicitud. Es útil para actualizar completamente un recurso existente.

PATCH: Se utiliza para enviar datos al servidor para que sean parcialmente modificados o actualizados en un recurso específico. A diferencia de PUT, PATCH no reemplaza completamente el contenido del recurso, sino que solo modifica las partes específicas indicadas en el cuerpo de la solicitud.

DELETE: Se utiliza para eliminar un recurso específico del servidor. Al enviar una solicitud DELETE, el servidor eliminará el recurso indicado en la URL de la solicitud.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método HTTP que se debe utilizar al enviar un formulario HTML, como uno donde ingreses tu usuario y contraseña en un sitio web, es el método POST. La razón principal es que el método POST se utiliza para enviar datos al servidor para su procesamiento, y en el caso de un formulario de inicio de sesión, los datos como el nombre de usuario y la contraseña deben enviarse al servidor de manera segura y no deben mostrarse en la URL ni en ningún otro lugar visible para el usuario. El método POST envía los datos en el cuerpo de la solicitud, lo que proporciona una capa adicional de seguridad en comparación con el método GET, que coloca los datos en la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Cuando se accede a una página a través de un URL en un navegador web, se utiliza el método HTTP GET de forma predeterminada. Este método se utiliza para solicitar datos de un recurso específico del servidor. Al ingresar una URL en la barra de direcciones del navegador y presionar Enter, el navegador envía una solicitud GET al servidor para recuperar la página web solicitada. El servidor luego responde con el contenido de la página, que el navegador interpreta y muestra al usuario. Este proceso es fundamental para la navegación web y es la forma más común en que se accede a páginas y recursos en Internet a través de los navegadores.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Cuando un servidor web devuelve una respuesta HTTP con el código 200, significa que la solicitud se ha procesado correctamente y que el servidor ha enviado el recurso solicitado al cliente de manera exitosa. El código de estado 200 en HTTP se conoce como "OK" o "Éxito". Indica que la solicitud del cliente ha sido recibida, entendida y aceptada, y que el servidor ha respondido correctamente con el contenido solicitado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. El error 404 indica que el recurso solicitado no fue encontrado en el servidor. Aunque puede haber varias razones para esto, como cambios en la estructura del sitio, enlaces rotos o recursos eliminados, es responsabilidad del desarrollador garantizar que el sitio esté disponible y funcione correctamente para los usuarios.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. El error 500, también conocido como "Error del servidor interno", indica que hubo un problema en el servidor que impidió que se procesara la solicitud del usuario correctamente. A diferencia de los errores 404, que generalmente se deben a recursos faltantes, los errores 500 son más graves y pueden deberse a problemas internos del servidor, como errores de programación, configuraciones incorrectas, problemas de permisos, entre otros.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando un atributo HTML5 está "depreciado" o "desaprobado" (deprecated), significa que se considera obsoleto o que su uso ya no se recomienda, aunque aún es compatible con los navegadores web por razones de retrocompatibilidad. La recomendación es dejar de usar estos atributos y migrar a alternativas más modernas y estándares.

font: En HTML4, font se usaba para cambiar el tamaño, color y tipo de fuente del texto. En HTML5, se recomienda usar CSS para estilizar el texto en lugar de etiquetas de formato inline.

center: En HTML4, center se usaba para centrar el contenido dentro de un elemento. En HTML5, se recomienda usar CSS para el centrado en lugar de una etiqueta específica de HTML.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 representa una evolución significativa sobre HTML 4 al introducir mejoras sustanciales en varios aspectos. Esto incluye una semántica mejorada con la incorporación de elementos semánticos como `header`, `footer` y `nav`, proporcionando una estructura más clara al contenido. Además, HTML5 ofrece compatibilidad nativa para multimedia a través de elementos como `audio` y `video`, simplificando la integración de contenido multimedia sin necesidad de complementos externos. También se mejoran los formularios con nuevos tipos de entrada y validación nativa, y se introducen capacidades gráficas avanzadas mediante SVG y la etiqueta `canvas`. Con la adición de nuevas APIs y funcionalidades, como la API de geolocalización y almacenamiento web, HTML5 permite desarrollar aplicaciones web más sofisticadas. Además, HTML5 incluye características que mejoran la accesibilidad y optimizan el SEO del sitio, lo que contribuye a una experiencia de usuario más rica y una mejor visibilidad en los motores de búsqueda.

¿Qué componentes de estructura y estilo tiene una tabla?

table: Es el elemento contenedor principal que define la tabla en HTML.

thead: Representa la fila o filas de encabezado de una tabla.

tbody: Representa el cuerpo de una tabla, que contiene las filas de datos.

tfoot: Representa el pie de una tabla, que puede contener resúmenes o totales de los datos.

Además de estos elementos principales, una tabla puede contener elementos de fila y celda:

tr: Representa una fila de la tabla.

th: Representa una celda de encabezado de tabla, que se utiliza para etiquetar las columnas o filas.

td: Representa una celda de datos de tabla, que contiene el contenido real de la tabla.

¿Cuáles son los principales controles de una forma HTML5?

input: Es el elemento más versátil y se utiliza para crear diversos tipos de campos de entrada, como texto, contraseña, correo electrónico, número, fecha, etc.

textarea: Permite a los usuarios ingresar texto multilínea.

select: Crea un menú desplegable para que los usuarios seleccionen una opción de una lista de opciones.

button: Crea un botón que puede usarse para enviar un formulario o realizar otras acciones específicas.

label: Se utiliza para asociar un texto descriptivo con un campo de formulario, lo que mejora la accesibilidad y la usabilidad del formulario.

fieldset y legend: Permiten agrupar varios campos de formulario relacionados y proporcionar una etiqueta descriptiva para el grupo.

input type="radio": Permite a los usuarios seleccionar una opción de un conjunto de opciones exclusivas.

input type="checkbox": Permite a los usuarios seleccionar una o varias opciones de un conjunto de opciones.

input type="file": Crea un campo de entrada para que los usuarios carguen archivos desde su dispositivo.

input type="submit" y input type="reset": Crean botones para enviar o restablecer un formulario, respectivamente.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información (SDLC, por sus siglas en inglés) es un modelo que describe las diferentes etapas que un sistema de información atraviesa desde su concepción hasta su retirada. Aunque puede variar según la metodología específica utilizada y las necesidades del proyecto, las etapas típicas del ciclo de vida de los sistemas de información incluyen:

Identificación de necesidades y oportunidades: Se identifican los problemas o las oportunidades que podrían resolverse mediante la implementación de un sistema de información.

Análisis de requisitos: Se recopilan y analizan los requisitos del sistema de información, incluidos los requisitos funcionales y no funcionales.

Diseño del sistema: Se diseña la arquitectura del sistema, definiendo la estructura, los componentes y la interfaz del sistema de información.

Desarrollo o implementación: Se construye o desarrolla el sistema de información según los requisitos y el diseño previamente establecidos.

Pruebas y validación: Se realizan pruebas para garantizar que el sistema funcione según lo esperado y cumpla con los requisitos establecidos. Esto incluye pruebas unitarias, de integración y de sistema.

Implementación y puesta en marcha: Se implementa el sistema en el entorno de producción y se pone en funcionamiento para su uso por parte de los usuarios finales.

Mantenimiento y actualización: Se proporciona soporte continuo, se realizan correcciones de errores y se realizan actualizaciones periódicas para mantener el sistema en funcionamiento y satisfacer las necesidades cambiantes de los usuarios.

Retirada o desuso: En esta etapa, el sistema se retira o se desactiva, ya sea porque ha sido reemplazado por un sistema nuevo o porque ya no es necesario.

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información (ISD, por sus siglas en inglés) se centra específicamente en el proceso de construir un sistema de información durante la fase de desarrollo del ciclo de vida del sistema. A menudo se basa en metodologías específicas de desarrollo de software, como el modelo en cascada, el modelo en espiral, la metodología ágil, entre otras. El ciclo de desarrollo de sistemas de información suele incluir etapas como:

Requisitos: Recopilación y análisis de requisitos del sistema.

Diseño: Diseño arquitectónico y detallado del sistema.

Implementación: Desarrollo y codificación del sistema.

Pruebas: Pruebas unitarias, de integración y de sistema para asegurar la calidad del sistema.

Implementación: Despliegue del sistema en el entorno de producción.

Mantenimiento: Soporte continuo, correcciones de errores y actualizaciones del sistema.

CSS

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

!important es una declaración que anula cualquier otra regla CSS que se aplique a un elemento, lo que puede hacer que tu código sea menos mantenible y más difícil de entender. Mi recomendación sobre el uso de !important en CSS es usarlo con moderación y solo cuando sea absolutamente necesario.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

La elección cuidadosa de una imagen de fondo en una página HTML es importante debido a que las imágenes de fondo grandes pueden aumentar significativamente el tiempo de carga de la página, especialmente en conexiones de internet más lentas o dispositivos móviles. Es importante seleccionar una imagen de fondo que tenga un tamaño de archivo lo suficientemente pequeño como para no ralentizar la carga de la página.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

al elegir las unidades de estilo CSS, es importante considerar la naturaleza del diseño y las necesidades de la aplicación. Para diseños responsivos y adaptativos, es probable que % sea la opción más adecuada para propiedades como anchos y alturas, mientras que px sigue siendo útil para propiedades donde se necesita un control preciso sobre las dimensiones. Las unidades pt son más adecuadas para contextos de impresión o cuando se requiere una medida absoluta independiente del tamaño de la pantalla.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

El uso de una versión minimizada del CSS mejora el rendimiento del sitio al reducir significativamente el tamaño del archivo CSS. Al eliminar espacios en blanco, saltos de línea, comentarios y otros caracteres no esenciales, la versión minimizada se descarga más rápidamente desde el servidor al navegador del usuario, requiere menos ancho de banda para su transferencia, utiliza menos memoria del navegador y es más propensa a ser almacenada en caché para visitas posteriores. Esto se traduce en tiempos de carga más rápidos y una experiencia de usuario más eficiente.
